<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser import payload → server records module | legacy snapshot 内容不可信，必须经过 DTO 校验与 current-user 绑定 |
| incoming local records → existing cloud records | 重复 `placeId` 可能导致错误覆盖或重复计数 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Elevation of Privilege | `POST /records/import` | mitigate | 端点强制 `SessionAuthGuard` + `@CurrentUser()`，不接受 body 中 owner 信息 |
| T-24-02 | Tampering | import payload de-duplication | mitigate | DTO 校验 + repository 先按 `placeId` 折叠 incoming，再按 `userId_placeId` 判断是否新建 |
| T-24-03 | Information Disclosure | cloud/local conflict handling | mitigate | 已存在云端记录保持 authoritative，不把其他账号或 legacy source 数据直接覆盖到当前账号 |
</threat_model>
