<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| previous authenticated snapshot → next authenticated/anonymous snapshot | 边界切换时最容易泄露上一账号 records |
| topbar identity UI → session state | 菜单/用户名展示必须与当前 snapshot 保持一致 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-10 | Information Disclosure | logout / switch-account boundary | mitigate | 所有边界切换先 reset records + selection + pending import state，再注入新 snapshot |
| T-24-11 | Repudiation | account switch feedback | mitigate | switch-account 明确给 info notice，避免静默切换让用户误解地图状态变化来源 |
| T-24-12 | Denial of Service | stale modal/menu state during boundary switch | mitigate | logout / switch-account 时关闭菜单和 pending dialogs，防止用户停留在过期交互面板上 |
</threat_model>
