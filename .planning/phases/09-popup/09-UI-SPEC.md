Popup-specific type rules:

State rules:

- Popup summary and drawer deep view must consume the same store state and copy source; no duplicated business template.
- Popup height target is medium-density: aim for 2-4 short text rows plus actions. If content needs sustained scrolling, hand off to drawer.
- Candidate search remains available inside popup/peek; it is not moved back to a side drawer.

## Placement And Responsive Contract

| Context | Contract | Source |
|---------|----------|--------|
| Desktop breakpoint | `min-width: 960px` keeps anchored popup as primary mode | existing breakpoint in `src/App.vue` and `src/components/WorldMapStage.vue` |
| Desktop anchor | Popup is rendered inside `WorldMapStage` and visually anchored to the selected marker, pending hit, or derived boundary center; never as fixed right rail or corner toast | `09-CONTEXT.md`, `09-RESEARCH.md` |
| Desktop size | Width clamps to `280px`-`360px`; padding `24px`; internal gap `16px`; anchor offset `12px` | `09-RESEARCH.md`, spacing/token alignment |
| Collision handling | Use viewport padding `16px`; allow flip/shift before fallback; keep popup fully readable without covering the anchor when possible | `09-RESEARCH.md` |
| Mobile / unsafe mode | If viewport `< 960px`, anchor is unsafe, or remaining height cannot hold the summary card, switch to bottom `peek` | `09-RESEARCH.md` |
| Peek sizing | Bottom peek spans full width minus `16px` side insets, sits above safe area, and caps height at `min(32rem, calc(100vh - 8.5rem))` | current drawer sizing in `src/components/PointPreviewDrawer.vue`, safe-area rule defaulted from research |

Anchor precedence:

1. Saved or draft marker anchor if a concrete point marker is visible.
2. Pending recognition hit anchor for pre-confirmation states.
3. Derived boundary visual center when a city boundary is the only stable visual reference.

## Interaction Contract

### Open / Close

- Opening the popup moves focus to the popup title or first interactive element, matching current drawer accessibility behavior.
- `Escape` closes the popup summary surface unless an inline destructive confirm row is open; in that case, `Escape` cancels the confirm row first.
- Closing the popup must call the same store action path as current drawer close; popup must not keep its own remembered city/highlight state.

### Focus And Keyboard

- Popup uses `role="dialog"` with `aria-modal="false"` to preserve map-stage context.
- Desktop popup keeps keyboard focus cycling inside the summary surface while open.
- All actionable controls must maintain a visible `2px` accent focus ring with `3px` offset.

### Destructive Actions

- `删除` and `隐藏` stay available inside popup for saved/seed states.
- Confirmation is lightweight: inline confirm row or second-tap confirm inside the popup action area.
- Do not use browser `window.confirm()` or a blocking full-screen modal in Phase 09.
- Destructive action stays visually separated from the primary CTA and never occupies the first slot in the action row.

### Highlight Synchronization

- Popup title, popup state, and boundary highlight must always refer to the same `cityId` / `boundaryId`.
- Close, switch, reopen, fallback, and reuse flows must leave highlight behavior entirely under existing store/overlay state.
- Popup dismissal never clears a saved boundary highlight unless the selected point itself changes or clears.

### Action Row Rules

- Action row order is `primary handoff/save`, `secondary edit/toggle`, `destructive`.
- Keep actions on a single row when space permits; wrap only on mobile peek.
- Do not place more than 3 visible actions in the default row; overflow actions must hand off to drawer.
