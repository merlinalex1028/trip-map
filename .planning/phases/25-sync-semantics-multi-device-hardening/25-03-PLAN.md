<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mutation result → user-facing notice | 同步成功、普通失败和需重登若混淆，用户会误解当前账号状态 |
| optimistic local state → server authoritative result | 需要保证失败时能恢复、成功时能稳定收敛 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-07 | Repudiation | sync feedback semantics | mitigate | 用统一 notice 文案明确区分成功、普通失败和需重新登录 |
| T-25-08 | Denial of Service | silent unilluminate failure | mitigate | 取消点亮普通失败必须 warning，不能只静默回滚 |
| T-25-09 | Information Disclosure | stale delete UI rollback | mitigate | 依赖 Plan 01 的幂等 delete 语义，让 stale delete 自然收敛到未点亮 |
</threat_model>
