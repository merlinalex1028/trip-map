<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| same authenticated user → foreground refresh path | 同 user refresh 若误走切账号路径，会清空当前上下文或制造错误 notice |
| browser focus / visibility events → network bootstrap | 焦点事件可能过于频繁，导致请求风暴或重复 state mutation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-04 | Denial of Service | foreground sync trigger | mitigate | refresh path 复用 in-flight promise / 只在 authenticated 状态触发，避免 focus 风暴 |
| T-25-05 | Information Disclosure | same-user refresh vs switch-account boundary | mitigate | 同 user refresh 只替换 records snapshot，不走 `resetTravelRecordsForSessionBoundary()` 与 switch notice |
| T-25-06 | Spoofing | refresh source of truth | mitigate | foreground sync 仍只信任 `/auth/bootstrap` 返回的 server snapshot 与当前 `sid` cookie |
</threat_model>
