<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| stale device snapshot → current-user delete endpoint | 已落后的本地状态可能把“已删除”误当成错误 |
| current-user mutation → other session bootstrap | 多 session 需要通过 server snapshot 收敛，而不是依赖各自本地缓存 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Tampering | stale delete semantics | mitigate | 同 user 缺失记录删除改成幂等成功，避免 stale device 回滚成旧状态 |
| T-25-02 | Information Disclosure | multi-session bootstrap consistency | mitigate | 用 `/auth/bootstrap` 作为当前账号权威 snapshot，并用 multi-session e2e 锁住 create/delete 后的收敛行为 |
| T-25-03 | Repudiation | sync contract ambiguity | mitigate | 用专门的 sync e2e 明确 create/delete/bootstrap 的预期结果，不让“404 是否算失败”继续模糊 |
</threat_model>
