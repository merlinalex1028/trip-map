# Phase 13 Plan 03: 生成并提交正式几何资产与 manifest 查询服务 Summary

**One-liner:** Versioned GeoJSON shards with injected renderableId + generated TypeScript manifest constant + manifest query service with WGS84 bounds validation for Beijing, Hong Kong, and California.

## What Was Built

### Task 1: 生成并提交版本化 manifest 与 shard 资产 (commit: 6e1c4f6)

Enhanced `build-geometry-manifest.mjs` to inject `renderableId`, `boundaryId`, and `datasetVersion` into each shard feature's properties at build time. Added `generateTypescriptManifest()` to emit `packages/contracts/src/generated/geometry-manifest.generated.ts` as a TypeScript constant. Ran `geo:build` to produce 6 official shard files under `apps/web/public/geo/2026-03-31-geo-v1/` and the generated manifest.

Updated `CityBoundaryFeatureProperties` in `apps/web/src/types/geo.ts` to include `renderableId: string`. Exported the generated manifest from `packages/contracts/src/index.ts`.

**Verification:** `pnpm --filter @trip-map/web run geo:build` + `pnpm --filter @trip-map/contracts test` — all 13 contracts tests pass.

### Task 2: 提供 manifest 查询服务并用真实 shard 建立验证基线 (commit: d2419db)

Created `apps/web/src/services/geometry-manifest.ts` exposing:
- `GEOMETRY_DATASET_VERSION = '2026-03-31-geo-v1'`
- `getGeometryManifestEntry(boundaryId)` — looks up from `GEOMETRY_MANIFEST` constant in contracts
- `listGeometryManifestEntriesByLayer(layer)` — filters by CN or OVERSEAS layer

Created `geometry-manifest.spec.ts` (9 tests) verifying: dataset version, entry lookup for beijing/hong-kong/california, null for unknown IDs, forbidden assetKey values, layer filtering, renderableId presence.

Created `geometry-validation.spec.ts` (9 tests) reading real shard files at `apps/web/public/geo/2026-03-31-geo-v1/` to verify:
- `renderableId`/`boundaryId`/`datasetVersion` in feature properties
- Representative click points within feature bounds (Beijing lat:39.9042, HK lat:22.3193, California lat:36.7783)
- WGS84 coordinate range assertions confirming GCJ-02 → WGS84 normalization

**Verification:** `pnpm --filter @trip-map/web test src/services/geometry-manifest.spec.ts src/services/geometry-validation.spec.ts` — all 18 tests pass.
