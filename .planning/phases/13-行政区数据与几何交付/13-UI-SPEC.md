## Phase Surface Contract

本阶段不新增独立页面、独立面板或新的地图主交互。实现必须限定在既有表面内：

| Surface | Contract |
|---------|----------|
| `WorldMapStage` | 保持现有世界地图舞台、pending marker、selected highlight 结构；只允许接入几何 shard 的加载与渲染状态，不得改成新的浏览式地图 UI |
| `MapContextPopup` | 继续作为摘要入口；新增几何状态反馈时，优先用 notice / badge / inline status，不能把 popup 变成完整编辑表单 |
| `PointPreviewDrawer` | 继续承担深度查看与编辑；如果几何缺失或加载失败，drawer 只展示状态说明与后续动作，不承担数据源调试面板 |
| Existing title shell | 保持 `PosterTitleBlock` 的现有海报式视觉，不因数据交付 phase 引入新的后台管理视觉语言 |

## Interaction Contract

| State | Contract |
|-------|----------|
| Geometry loading | 当 `server` 已返回 `geometryRef` 但 shard 尚未解析完成时，在当前 popup 或 drawer 内显示单条非阻塞状态文案；不得清空已识别地点标题、类型、副标题 |
| Geometry ready | 成功加载边界后，边界高亮立即替换 pending/placeholder 状态；popup anchor 以真实边界中心点或既有 boundary anchor 规则为准，不能因加载完成产生明显跳闪 |
| Geometry missing | 当 manifest 明确无对应 renderable geometry 时，沿用当前 unsupported tone，展示“仍可保存 canonical 地点信息”的说明；不得伪造错误边界或静默降级为另一地点 |
| Geometry failed | 当 shard 请求或解析失败时，保留地点摘要与已有 CTA，可提供 `稍后重试` 级别动作；不得把失败文案写成地点识别失败，避免混淆 resolve 与 geometry delivery |
| Layer identity | 中国市级与海外一级行政区必须继续通过现有类型标签与副标题清晰区分；Phase 13 不允许为了几何统一而把两类地点重新伪装成“城市” |
| Destructive continuity | 现有删除/隐藏确认链路保持原样；几何加载失败不触发任何 destructive confirmation |

## Visual Contract

| Topic | Contract |
|-------|----------|
| Primary focal point | 主视觉焦点始终是当前选中行政区边界高亮；popup / drawer 内的 geometry 状态块只作为次级反馈，不得反客为主压过地图主体状态 |
| Status hierarchy | 几何状态优先使用现有 `notice` / `badge` / wash background 体系表达，不新增 toast flood、全屏遮罩或 debug console 弹层 |
| Highlight color | 当前选中边界继续使用 `selected` 粉色系；已保存边界继续使用 `saved` 蓝色系；geometry-missing 使用 fallback 灰蓝系，和现有 `boundarySupportNotice` 语义一致 |
| Surface density | popup 与 drawer 内新增内容最多增加一个状态块和一个次级动作，不得挤压当前标题、副标题、候选列表与编辑入口的可读性 |
| Motion | 继续使用现有 quick/emphasis 节奏；几何加载完成可复用轻量过渡，但不得新增地图大范围飞行动画或复杂骨架屏 |
