<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `/auth/bootstrap` -> Pinia stores | Server-authoritative travel-record metadata crosses into client state during restore or same-user refresh |
| `mapPointsStore.travelRecords` -> `StatisticsPageView` watcher | Client route code decides when refreshed metadata invalidates cached server stats |
| `StatisticsPageView` -> `/records/stats` | Client requests authenticated stats; response must not overwrite newer session/request state |
| Same-user refresh -> session boundary state | Metadata refresh must not be treated as an account switch or unauthorized session boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Tampering | `StatisticsPageView.vue` `travelRecordRevision` | mitigate | Include `parentLabel`, `displayName`, `typeLabel`, and `subtitle` in the watched revision so authoritative metadata changes cannot leave stale country/completion stats cached. |
| T-31-02 | Information Disclosure | `statsStore.fetchStatsData()` stale responses | mitigate | Preserve `boundaryVersion` and `activeRequestId` guards by not editing `apps/web/src/stores/stats.ts`; run `src/stores/stats.spec.ts` to prove stale responses remain ignored. |
| T-31-03 | Denial of Service | metadata refresh watcher | mitigate | Preserve `pendingRefreshAfterLoad`; route test asserts metadata changes during loading queue one follow-up fetch instead of overlapping request loops. |
| T-31-04 | Spoofing / Elevation of Privilege | same-user authoritative refresh | mitigate | Add `auth-session.spec.ts` regression asserting same-user metadata refresh does not call `resetTravelRecordsForSessionBoundary()` and leaves `boundaryVersion` unchanged. |
| T-31-05 | Repudiation | invisible metadata refresh | accept | No new audit log is added because Phase 31 is a route-local UI freshness fix with no new user action or server write path; correctness is proven through automated regressions. |
</threat_model>
