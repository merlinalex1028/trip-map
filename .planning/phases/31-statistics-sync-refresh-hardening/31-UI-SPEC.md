# Phase 31 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

Phase 31 is a frontend consistency hardening phase, not a redesign phase. Preserve the existing statistics page visual language and interaction model. Do not introduce new components, new visual states, new copy, new colors, new motion, or a new design system unless required to preserve current behavior.

Primary visual anchor remains the existing statistics overview panel first, followed by the three `StatCard` cards; metadata refresh must not introduce a new focal point.

| Property | Value |
|----------|-------|
| Tool | none — custom Vue + Tailwind CSS v4 utilities; no `components.json` detected |
| Preset | not applicable |
| Component library | none — use existing local Vue components only |
| Icon library | none for this phase |
| Font | `Nunito Variable`, `Noto Sans SC`, `PingFang SC`, `Hiragino Sans GB`, `Microsoft YaHei`, sans-serif |

Source notes: design tokens come from `apps/web/src/styles/tokens.css`; current page/component patterns come from `apps/web/src/views/StatisticsPageView.vue` and `apps/web/src/components/statistics/StatCard.vue`; Phase 31 scope comes from `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`, and `31-RESEARCH.md`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Existing inline gaps, chip/card micro-spacing |
| sm | 8px | Compact vertical spacing, `space-y-2`, small control gaps |
| md | 16px | Default card/grid gap, `gap-4`, compact panel padding |
| lg | 24px | Section/card padding where already present, `p-6` equivalent |
| xl | 32px | Layout gutters and major page spacing where already present |
| 2xl | 48px | Major section breaks only if existing layout already uses them |
| 3xl | 64px | Page-level spacing only if existing layout already uses it |

Exceptions: preserve existing statistics page values that are already part of the UI contract: `min-h-11` / 44px for interactive targets, 20px panel padding via `p-5`, 12px overview row gap via `gap-3`, 28px state card radius, 24px overview card radius, and full-pill radius for chips/buttons. Do not add new spacing exceptions in Phase 31.

## Color

Preserve the existing soft pastel statistics language. Phase 31 is about data freshness and synchronization, so visual color behavior must remain unchanged across loading, anonymous, error, empty, and populated states.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAFA` / `#FFF5F5` gradient | Page background and dominant soft surface atmosphere |
| Secondary (30%) | `rgba(255,255,255,0.72–0.95)`, `#fdf5ff`, `#fffaf2`, `#eef8ff` | Cards, state panels, stat-card surfaces, overview panel |
| Accent (10%) | `#f48fb1`, `#ff78ad`, `#ffd7ea` | Login CTA accent, selected/highlight wash, decorative warm gradients already present |
| Destructive | `#c86464` | Destructive actions only; no destructive actions are in Phase 31 scope |

Accent reserved for: the existing anonymous-state `立即登录` CTA, existing warm stat-card gradients, and existing selected/highlight decorative washes. Do not use accent to indicate background refresh, successful sync, queued refresh, or metadata-only changes.

## Interaction Contract

Phase 31 must preserve the existing state machine and make synchronization changes invisible unless a normal existing state already applies.

| State / Interaction | Contract |
|---------------------|----------|
| Initial restoring or loading | Continue showing the existing skeleton/loading panel with no new spinner, toast, banner, or progress text |
| Anonymous | Continue showing the existing login prompt and `立即登录` button; clicking opens `authSessionStore.openAuthModal('login')` |
| Error | Continue showing `统计数据加载失败`, existing body copy, and `重新加载统计`; retry continues to call `statsStore.fetchStatsData()` |
| Empty | Continue showing `还没有旅行数据`, existing guidance, and `去地图添加旅行` link to `/` |
| Populated | Continue showing overview copy, summary chip, and three `StatCard` cards in the current order: total trips, unique places, visited countries/regions |
| Metadata-only authoritative refresh | Trigger the needed stats refresh without changing layout, copy, focus, scroll position, route, or visible loading treatment unless the page is already in the normal loading state |
| In-flight refresh coalescing | Queue at most one follow-up refresh through the existing pending-refresh model; do not flash empty/error/populated states between same-session refreshes |
| Same-user session refresh | Apply authoritative metadata and update statistics without resetting the session boundary UI or showing anonymous state |

Accessibility invariants: preserve 44px minimum interactive targets, button/link semantics, existing route link behavior, and readable contrast from current tokens. Do not add automatic focus movement, announcements, or modal behavior for invisible metadata refreshes.

## Copywriting Contract

Do not change user-facing copy in Phase 31. The goal is consistency hardening with no visual or copy drift.

| Element | Copy |
|---------|------|
| Primary CTA | `重新加载统计` for error retry; `立即登录` remains the anonymous-state login CTA |
| Empty state heading | `还没有旅行数据` |
| Empty state body | `先回到地图点亮一个去过的地点吧。保存后，这里会出现你的第一条统计数据。` |
| Error state | `统计数据加载失败` + `无法获取旅行统计，请稍后重试。` |
| Destructive confirmation | none — Phase 31 has no destructive actions |

Additional fixed labels: keep `旅行统计概览`, `总旅行次数`, `已去过地点数`, `已去过国家/地区数`, `次旅行`, `个地点`, and `个国家/地区` unchanged.
