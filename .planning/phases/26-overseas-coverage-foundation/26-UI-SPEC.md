## Phase UI Contract

| Surface | States | Contract |
|---------|--------|----------|
| 海外地点 popup 主卡片 | `detected-preview` / `view` / `saved` | 继续复用现有 `PointSummaryCard` 云朵卡片，不新增海外专用面板。单一 canonical 命中时直接显示主标题、类型 pill、副标题和右上角主按钮。主标题固定使用英文 canonical `displayName`；类型 pill 固定使用中文 `一级行政区` 体系；副标题固定使用 `英文国家名 · 中文类型说明`。已保存记录回放时，三段文本必须与写入 `TravelRecord` 时完全一致，不允许前端按 `placeId` 二次拼接。 |
| 未支持地区 popup 说明 | `unsupported-detected` | 对已识别但暂不支持点亮的海外区域，只在当前 popup 内给出说明，不弹全局 `interactionNotice`。说明块放在卡片正文第一条 notice，按钮继续留在标题区但保持禁用。说明文案固定为两句：`已识别到该地区。` + `当前暂不支持点亮，请优先在日本、韩国、泰国、新加坡、马来西亚、阿联酋、澳大利亚、美国的一级行政区使用。`；如果已有更具体的地区名，可替换首句中的“该地区”为具体名。 |
| 多候选确认列表 | `candidate-select` | 仅在 `ambiguous` 时进入候选确认，不把海外点击默认改成二次确认流程。候选列表最多展示 `3` 个项，推荐项使用 accent 描边和更亮底色，其余候选维持 secondary/fallback 色阶。每项必须同时显示 `displayName`、`typeLabel`、`subtitle` 与状态提示；按钮文案固定为 `确认地点`。已存在记录的候选优先显示 `已存在记录` 状态，不再重复强调“推荐”。 |
| 点亮按钮语义 | `available` / `saved` / `disabled-unsupported` / `pending` | 可点亮时沿用单一主按钮，文案保持现有 `点亮` / `已点亮` 切换；不新增“海外专用”CTA 文案。未支持地区仍显示同一位置按钮，但以禁用态表达不可操作，并通过 `title`/`aria-label` 复述 `该地点暂不支持点亮`。pending 态只禁用当前按钮，不冻结整个 popup。 |
| 边界缺失提示 | `boundary-missing` | 若记录文本可保存但边界暂不可高亮，说明块必须作为第二条 notice 出现在正文内，文案保持 `当前地点暂不支持边界高亮，将仅保存 canonical 地点身份与文本信息`。此提示优先级低于 unsupported 说明，不可挤占第一条 notice。 |

### Visual Priority

- 第一视觉焦点始终是 popup 顶部的英文 canonical 标题，其次是右上角 `点亮 / 已点亮` 主按钮。
- 未支持地区时，第一条正文 notice 是主要解释层；用户不应先看到全局 toast，再回到 popup 才理解原因。
- 候选确认态时，推荐项边框和按钮高亮可以吸引注意，但不得压过候选标题本身。

## Motion And Accessibility

- 继续沿用现有动效 token：`var(--motion-quick)` 用于按钮、候选卡、notice 细微过渡；`var(--motion-emphasis)` 只用于 popup 进出或较强调的浮层切换。
- 所有 popup 操作按钮、候选项按钮最小点击高度保持 `44px`；现有 `min-h-11` 约束继续有效。
- 未支持地区的禁用按钮必须保留可见标签，不可隐藏按钮后只留一段文案；键盘焦点进入按钮时应能通过 `title` 或 `aria-label` 得知 `该地点暂不支持点亮`。
- `candidate-select` 列表保持按钮语义和可见 focus ring；推荐项只通过额外视觉强调区分，不可只靠颜色区分状态。
- `prefers-reduced-motion: reduce` 下，hover 放大与位移动效应被移除，仍保持层级、边框和阴影可辨识。

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | `4px` | icon 与标签微间距、notice 内局部补白（source: existing `--space-xs`） |
| sm | `8px` | 类型 pill 与标题邻接、候选元信息垂直间距（source: existing `--space-sm`） |
| md | `16px` | notice 内边距、正文默认块间距、按钮与说明文案分隔（source: existing `--space-md`） |
| lg | `24px` | popup card 主 padding、标题区与正文区间距（source: existing `--space-lg`） |
| xl | `32px` | 桌面 popup 内部大块留白与地图舞台呼吸感（source: existing `--space-xl`） |
| 2xl | `48px` | 大屏场景中的主舞台分段与弹出层外部安全边距（source: existing `--space-2xl`） |
| 3xl | `64px` | 页面级留白；Phase 26 不新增超过此级别的新间距 token（source: existing `--space-3xl`） |

Exceptions: `44px` 作为主按钮、候选按钮、登录升级按钮等所有交互控件的最小触达高度；候选卡内部 outline 偏移保持 `4px`，不额外扩展点击区域（source: existing `min-h-11` + current popup styles）

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAFA` | 页面背景、地图外围底色、popup 外层整体氛围基底（source: existing `--color-page`） |
| Secondary (30%) | `#FDF5FF` | popup 主卡片、候选列表卡片基底、类型 pill 的柔和表面（source: existing `--color-surface`） |
| Accent (10%) | `#F48FB1` | 可点亮主 CTA、推荐候选描边、focus ring、推荐态强调标签（source: existing `--color-accent`） |
| Destructive | `#C86464` | 仅用于已有 `取消点亮/退出登录` 等破坏性语义；Phase 26 的 unsupported 说明不得借用 destructive 红（source: existing `--color-destructive`） |

Accent reserved for: 可操作的 `点亮` 主按钮、候选确认中的推荐项高亮、键盘 focus ring、交互性 status cue。不得用于 unsupported notice 背景、普通说明文字、整张 popup 底色或所有按钮。

补充色约束：已保存/已存在记录态优先继续使用 `#84C7D8` 及其浅色 wash；未支持或 fallback 说明优先使用中性/灰蓝 notice 色阶，不与 destructive 语义混淆（source: existing `--color-secondary`, `--color-state-fallback`）。
