    新建 `apps/web/src/services/api/canonical-places.ts`，只暴露两个函数：
    - `resolveCanonicalPlace(payload: ResolveCanonicalPlaceRequest): Promise<CanonicalResolveResponse>`
    - `confirmCanonicalPlace(payload: ConfirmCanonicalPlaceRequest): Promise<CanonicalResolveResponse>`

    这两个函数必须分别使用 `createApiUrl('/places/resolve')` 和 `createApiUrl('/places/confirm')`，并复用和 `phase11-smoke.ts` 一致的 `parseJsonResponse<T>` 风格错误处理。

    - `status === 'ambiguous'` 时把 server candidates 交给 store 的 canonical pending 入口
    - `status === 'failed'` 时只调用 `setInteractionNotice()` / `finishRecognition()`，不要创建任何 draft 或 fallback point
    - 候选确认改走 `confirmCanonicalPlace()`，不要再本地 `confirmPendingCitySelection()` 里生成 authoritative candidate 集
