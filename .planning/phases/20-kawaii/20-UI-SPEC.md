## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, sparkle offsets, tight inline chip spacing |
| sm | 8px | Compact inline padding, badge-to-label separation |
| md | 16px | Default element spacing inside cards and shell rows |
| lg | 24px | Card padding, popup body padding, section gaps |
| xl | 32px | Map shell breathing space, large card separation |
| 2xl | 48px | Major vertical breaks between fixed chrome and content on desktop |
| 3xl | 64px | Page-level breathing space on wide layouts only |

Exceptions: interactive buttons and candidate cards keep a minimum 44px hit area even when visual padding is smaller; non-interactive status pills may sit at 28-32px height; topbar stays `h-14 md:h-16` and must not grow taller to satisfy spacing.

Source: spacing tokens come from `apps/web/src/styles/tokens.css`; topbar height guardrail comes from `App.vue` and Phase 19/20 context.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAFA` | Page background, app shell, map stage backdrop, large negative space |
| Secondary (30%) | `#FDF5FF` | Popup/card interiors, notice surfaces, topbar pills, soft containers |
| Accent (10%) | `#F48FB1` | Primary CTA fill, current-selection highlight, recommended candidate emphasis, primary focus ring |
| Destructive | `#C86464` | Destructive actions only |

Accent reserved for: unsaved `点亮` CTA, currently selected warm-pink state, recommended candidate emphasis, and small-count attention highlights inside the popup card. Do not use accent pink for every hoverable control.

Supplemental semantic color contract: saved / 已点亮 state stays blue-mint (`#84C7D8` wash on `#DFF5FB`); fallback / warning notices stay blue-gray (`#B8C6D9` family) rather than switching to destructive red.

Source: color families inherit from `apps/web/src/style.css` and `apps/web/src/styles/tokens.css`; selected vs saved semantics are locked by Phase 10 / Phase 15 / Phase 20 context.

| Primary CTA | `点亮` |

Supplemental CTA copy: candidate confirm action keeps `确认地点`; saved-state toggle label keeps `已点亮`. These labels are already present in `PointSummaryCard.vue` and should not be改写成更长、更解释型的按钮文案。

## Surface Contract

| Surface | Visual Contract | Interaction Contract |
|---------|-----------------|----------------------|
| App shell / topbar | Keep current thin-shell layout. Topbar remains fixed `h-14 md:h-16`, uses pill sublabels and soft white border/shadow only. Sweetness comes from color, radius, and pills, not extra vertical height. | No hover scale on the fixed topbar container. Notice pill may fade/slide softly, but top chrome stays stable. |
| Map shell | Keep rounded outer stage with soft border and roomy padding. It is a framed canvas, not a cloud card. | No transform on the map shell or any ancestor of the Leaflet container. |
| `MapContextPopup.vue` outer shell | Outer layer stays light: 1px border, light blur, light shadow, arrow inherits same light shell. No `border-4`, no heavy cloud shadow here. | Outer anchored shell does not scale on hover. It must remain geometrically stable so popup anchoring does not drift. |
| `PointSummaryCard.vue` inner card | This is the cloud card. Apply `rounded-3xl`, `border-4 border-white`, soft pastel gradient, and the main floating shadow. Padding is `p-6` minimum with `gap-4` minimum between content groups. | If hover motion is used on the popup/card, it applies here only, not to the anchored outer shell. Hover lift stays subtle. |
| Primary CTA | Fully pill-shaped, accent-forward, sweetest surface in the popup. Use color-matched pink shadow, clear white edge, and stronger contrast than any other control. Minimum height 44px. | `duration-300 ease-out`; hover `scale-105 -translate-y-1`; active `scale-95`; disabled state drops motion and contrast. |
| Secondary CTA / candidate action | Still pill / soft rounded, but calmer than primary CTA. Use white / lilac / saved-blue surfaces with lighter shadow and lower contrast. Minimum height 44px. | Same `300ms ease-out` family. Hover may lift and scale lightly; active press exists but must feel softer than primary CTA. |
| Info badge / type pill / notice capsule | Pill language stays, but badges read as labels, not buttons. Use lighter borders, lighter fill, and smaller height than CTAs. | Non-interactive badges have no scale or active state. Interactive pills may keep color/shadow feedback only. |
| Pending marker / boundary / map-hit surfaces | Preserve current restrained map semantics. Warm pink for selected/current, blue-mint for saved. No candy bounce on geospatial layers. | No scale, no translate, no animation that changes hit geometry. Motion may only be opacity/glow-level and must not affect coordinates. |

Source: Phase 20 context D-05 to D-16, `App.vue`, `MapContextPopup.vue`, `PointSummaryCard.vue`, `LeafletMapStage.vue`.

## Motion Contract

- All interactive UI surfaces in scope use `300ms ease-out` as the default transition timing.
- Allowed hover combo for buttons, candidate cards, and inner cloud card: `scale-105` + `-translate-y-1` + shadow increase.
- Allowed active combo for buttons only: `scale-95` with no extra bounce and no spring overshoot.
- Candidate cards and popup inner card use lighter motion than the primary CTA; motion should read as airy lift, not cartoon jump.
- `MapContextPopup.vue` outer shell, `.leaflet-map-stage`, `.leaflet-map-stage__map`, and popup anchor parents must never receive `transform`.
- `prefers-reduced-motion` fallback keeps color, border, and shadow transitions, but removes scale/translate.

Source: `STYLE-03` to `INTER-03`, Phase 20 context D-13 to D-16, and research pitfalls on Leaflet transform safety.
