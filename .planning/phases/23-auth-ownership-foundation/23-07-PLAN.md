<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser request → SessionAuthGuard | 客户端可以伪造 body/query/header owner 字段，但不能被当作真实当前用户 |
| SessionAuthGuard → downstream controllers | 后续 `/records` 等受保护路由会直接信任 guard 注入的 `request.authUser` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-09 | Spoofing | `SessionAuthGuard` current-user recovery | mitigate | guard 仅信任 cookie sid 和服务端 session 查找结果，忽略 bearer/body/query/header owner 线索 |
| T-23-10 | Elevation of Privilege | downstream protected routes | mitigate | `CurrentUser` 只读取 guard 已写入的 `request.authUser`；未通过 guard 的请求直接 `401` |
| T-23-11 | Tampering | module wiring | mitigate | `AuthModule` 显式导出 `AuthService` 与 `SessionAuthGuard`，后续模块通过 import 复用同一实现，不复制解析逻辑 |
</threat_model>
