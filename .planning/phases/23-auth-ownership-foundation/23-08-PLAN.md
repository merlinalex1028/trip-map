<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| controller input → `RegisterDto` | 浏览器提交的 `username/email/password` 不可信，必须在入口完成规范化与验证 |
| `AuthService.register` → Prisma writes | 两次独立写库之间若无事务保护，会产生账号残留和会话不一致 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-15 | Tampering | `AuthService.register` / Prisma user+session writes | mitigate | 用 `createUserWithSession()` + Prisma `$transaction` 把 `User` 与 `AuthSession` 写入收口成单原子单元 |
| T-23-16 | Denial of Service | register failure path | mitigate | session 创建失败时整次回滚并返回错误，避免数据库留下半成品账号造成后续重复注册/登录异常 |
| T-23-17 | Spoofing | `username` identity display | mitigate | 在 `RegisterDto` 里 trim 后再做 `2..32` 校验，拒绝空白用户名进入持久化与顶栏身份展示链路 |
</threat_model>
