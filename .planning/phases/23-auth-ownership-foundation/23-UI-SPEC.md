## Phase UI Contract

| Surface | States | Contract |
|---------|--------|----------|
| 顶栏账号 chip | `anonymous` / `authenticated` | 入口固定放在顶栏右侧空位，不新增独立账号页。未登录时显示单个 pill 按钮 `登录 / 注册`；已登录时显示用户名 chip。用户名只占一行，超出后省略，chip 最大宽度移动端 `9rem`、桌面端 `12rem`。authenticated 状态允许带 1 个小尺寸状态点或下拉箭头，但不新增头像体系。 |
| 账号下拉菜单 | `closed` / `open` | 仅在 `authenticated` 状态出现，锚定账号 chip 下沿右对齐展开。菜单宽度固定在 `15rem` 到 `17rem`，内容顺序固定为：用户名、邮箱、分隔线、`退出登录`。不允许加入设置页、设备管理、占位入口。关闭方式只允许外部点击、`Esc`、执行退出后自动关闭。 |
| 认证弹层 | `login` / `register` / `submitting` / `error` | 未登录时点击 chip 打开同一个居中弹层，内部用两个 tab 切换。首次打开默认停在 `登录` tab；tab 不跳页、不分步。注册表单字段固定为 `用户名 + 邮箱 + 密码`；登录表单固定为 `邮箱 + 密码`。主按钮全宽，文案按当前 tab 切换。字段错误就近显示在输入框下方，错误文案保持单句，不引入帮助链接。成功后自动关闭弹层、返回焦点到顶部入口，并立即刷新账号态与地图记录视图。 |
| 会话恢复蒙层 | `restoring` / `dismissed` | App 启动时保留现有顶栏与主舞台，只在地图主舞台上方覆盖柔和蒙层，不切全页 loading。蒙层必须保留地图轮廓可见性，建议 18% 到 24% 遮罩强度 + 轻度模糊；中间放一块小型浮层，文案使用 `正在恢复账号会话` + `正在载入你的云端旅行记录。`。恢复成功后直接淡出；恢复失败或 session 失效后淡出并落 warning notice。 |
| 全局 notice pill | `info` / `warning` | 继续复用现有顶部居中 notice。`info` 只用于已退出、已切回匿名浏览、已打开已有记录这类可恢复反馈；`warning` 用于登录失败、注册失败、会话恢复失败、session 失效。notice 自动消失时间保持 `2600ms`，不叠加多个 notice；新 notice 到来时覆盖旧 notice。 |
| 地图记录视图切换 | `anonymous` / `authenticated` / `logouting` | 切换到登录态后，地图记录必须先清空匿名上下文，再加载当前账号 records；切回匿名态时，必须先清空上一账号记录，再回到未登录视图。整个切换过程中不得闪现“空地图即空数据”的假状态；有加载过程时必须使用舞台蒙层覆盖。 |

## Motion And Accessibility

- 所有新增动效只使用现有时长 token：`var(--motion-quick)` 用于 chip、menu、tab；`var(--motion-emphasis)` 用于 modal 与 restoring overlay。
- 账号 chip、下拉菜单项、tab trigger、主提交按钮最小可点击高度为 `44px`；输入框最小高度为 `44px`。
- 认证弹层使用语义化 `dialog` + `aria-labelledby`，打开后焦点进入当前 tab 标题或首个输入框；关闭后焦点返回触发 chip。
- tab 切换使用 `tablist` 语义并支持键盘左右切换；字段级错误使用 `aria-describedby` 关联错误文本。
- 下拉菜单与弹层都支持 `Esc` 关闭；当请求进行中时禁止重复提交，但不强制阻断 `Esc` 以外的浏览。
- restoring overlay 是视觉 loading，不得把地图容器从 DOM 中移除；底层地图与 shell 结构持续存在。

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 输入框内图文微间距、状态点与文本间距（source: existing `--space-xs`） |
| sm | 8px | 字段标签与错误提示间距、chip 内边距、菜单项内部紧凑留白（source: existing `--space-sm`） |
| md | 16px | 表单字段垂直间距、默认卡片内边距、菜单主体留白（source: existing `--space-md`） |
| lg | 24px | 认证弹层主体 padding、分组区块间距（source: existing `--space-lg`） |
| xl | 32px | 弹层主内容与标题区分隔、舞台蒙层浮层留白（source: existing `--space-xl`） |
| 2xl | 48px | 桌面端弹层外边距、主舞台大块间距（source: existing `--space-2xl`） |
| 3xl | 64px | 页面级呼吸感与桌面端顶部到主舞台的稳定留白（source: existing `--space-3xl`） |

Exceptions: `44px` 作为账号 chip、菜单项、tab、输入框和主按钮的最小触达高度；顶栏继续沿用现有 `56px` mobile / `64px` desktop shell 高度（source: existing `App.vue` + phase default）

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAFA` | 页面背景、地图外层 canvas、restoring overlay 的底色基底（source: existing `--color-page`） |
| Secondary (30%) | `#FDF5FF` | 认证弹层壳层、账号下拉菜单、chip 浅底、表单卡片面（source: existing `--color-surface`） |
| Accent (10%) | `#F48FB1` | `创建账号/登录账号` 主按钮、激活 tab、authenticated chip 强调、focus ring、restoring loader 高亮（source: existing `--color-accent`） |
| Destructive | `#C86464` | `退出登录` 文本与 hover 态，仅用于退出动作，不参与主 CTA（source: existing `--color-destructive`） |

Accent reserved for: 主提交按钮、当前激活 tab、已登录账号 chip 的强调态、输入焦点 ring、会话恢复中的轻量进度提示；不得用于所有按钮、普通文字链接或整个菜单背景。
