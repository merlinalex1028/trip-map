在 `apps/server/prisma/schema.prisma` 现有模型之后追加三个新模型：

```prisma
model User {
  id           String   @id @default(cuid())
  username     String
  email        String   @unique
  passwordHash String
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt

  sessions     AuthSession[]
  travelRecords UserTravelRecord[]
}

model AuthSession {
  id        String   @id @default(cuid())
  userId    String
  expiresAt DateTime
  createdAt DateTime @default(now())

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId])
  @@index([expiresAt])
}

model UserTravelRecord {
  id             String   @id @default(cuid())
  userId         String
  placeId        String
  boundaryId     String
  placeKind      String
  datasetVersion String
  displayName    String
  regionSystem   String?
  adminType      String?
  typeLabel      String?
  parentLabel    String?
  subtitle       String
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([userId, placeId])
  @@index([userId])
}
```

**关键设计说明：**
- `User.email` 使用 `@unique` 确保邮箱唯一（per AUTH-01）。
- `User.username` 不做唯一约束，只限制长度在 DTO 层。
- `AuthSession` 使用 `cuid()` 作为 session token（即 `sid` cookie 值），per D-14 采用服务端 session 表。
- `UserTravelRecord` 使用 `@@unique([userId, placeId])` 复合唯一键，允许不同用户点亮同一 canonical place（per SYNC-01/SYNC-02）。
- 旧 `TravelRecord` 和 `SmokeRecord` 保持不动，legacy 数据隔离留给 Phase 24。
- `AuthSession.expiresAt` 用于会话过期判断，maxAge 30 天。
- `onDelete: Cascade` 确保删除用户时级联清理 sessions 和 records。
