<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client → `/records` | 客户端可伪造 place metadata，但不能决定 owner |
| records service → database | 应用层 access control 必须与数据库复合唯一键一致 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-06 | Elevation of Privilege | `/records` current-user endpoints | mitigate | 所有受保护 routes 强制 `SessionAuthGuard + @CurrentUser()`；service/repository 接口一律显式接收 `userId` |
| T-23-07 | Tampering | `POST /records` payload | mitigate | 服务器忽略任何 owner 概念，仅用 session `userId` 做 `upsert(userId, placeId)`；客户端 DTO 不新增 `userId` |
| T-23-08 | Information Disclosure | `GET /records` | mitigate | repository 统一 `where: { userId }`；跨账号同 placeId 只按复合唯一键命中当前用户记录 |
</threat_model>
