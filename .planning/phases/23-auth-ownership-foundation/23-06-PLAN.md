<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser cookie → bootstrap/guard | 未受信任客户端提交 `sid`，服务端必须验证 session 是否存在且未过期 |
| bootstrap response → web app | 前端会直接信任 bootstrap 返回的用户和 records 快照，服务端必须确保来源于当前 session |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-06 | Spoofing | `sid` cookie session | mitigate | 仅信任服务端 `AuthSession` 表命中的未过期 `sid`；无效/过期 sid 统一回落到 `authenticated: false` |
| T-23-07 | Information Disclosure | bootstrap session recovery | mitigate | 失效 session 统一返回 `authenticated: false` 并清理 cookie，避免把内部异常暴露给客户端 |
| T-23-08 | Tampering | bootstrap `records` snapshot | mitigate | bootstrap 只从当前 session 对应用户加载 `userTravelRecord` 快照，不回退到匿名全局 records |
</threat_model>
