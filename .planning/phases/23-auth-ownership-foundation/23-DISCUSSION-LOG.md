## 账号入口

| Option | Description | Selected |
|--------|-------------|----------|
| 顶栏右侧账号 chip + 下拉菜单 | 贴合现有 topbar，身份入口稳定，不打断地图主舞台 | ✓ |
| 顶栏右侧单按钮 + 弹层 | 更聚焦，但查看当前身份要多一步 | |
| 顶栏右侧账号卡片常驻展开 | 信息最直观，但会明显挤占顶栏空间 | |
| 其他 | 自定义布局方案 | |

**User's choice:** 顶栏右侧账号 chip + 下拉菜单  
**Notes:** 用户明确要求 chip 主展示为用户自定义用户名，且需要限制字数；无需考虑移动端方案。下拉菜单保持最小闭环，只放用户名、邮箱和退出。

## 注册与登录表单节奏

| Option | Description | Selected |
|--------|-------------|----------|
| 一个弹层，两种 tab：注册 / 登录 | 统一入口，切换直接，最轻量 | ✓ |
| 先输邮箱，再判断走注册还是登录 | 更像 onboarding，但分支与文案更复杂 | |
| 注册和登录分成两个独立弹层 | 可实现，但体验更笨重 | |
| 其他 | 自定义认证节奏 | |

**User's choice:** 一个认证弹层内使用 `注册 / 登录` 两个 tab 切换  
**Notes:** 注册表单固定为 `用户名 + 邮箱 + 密码`；登录表单固定为 `邮箱 + 密码 + 提交`；不做记住我、忘记密码或注册后再登录一次。注册 / 登录成功后自动关闭弹层，并同步刷新顶栏账号态与地图记录。

## 会话恢复时的界面表现

| Option | Description | Selected |
|--------|-------------|----------|
| 应用启动时做一次轻量恢复态，恢复完成后再进入正常地图交互 | 避免“空状态闪一下”的错觉 | ✓ |
| 完全静默恢复 | 最顺，但容易短暂展示错误空态 | |
| 地图先显示，但顶部固定出现恢复横幅 | 解释性更强，但主舞台仍可能误导 | |
| 其他 | 自定义恢复节奏 | |

**User's choice:** 轻量恢复态  
**Notes:** 恢复态保留 app shell 和顶栏，只在地图主舞台加柔和 loading 蒙层，不切换成独立全页 loading。若恢复失败，则结束恢复态、回到未登录视角，并给出明确提示。

## 多设备会话策略

| Option | Description | Selected |
|--------|-------------|----------|
| 允许多设备同时保持登录 | 最符合跨设备同步目标，当前设备退出只影响当前会话 | ✓ |
| 新设备登录后自动让旧设备下线 | 更强控制，但与多设备同步预期相冲突 | |
| 允许多设备登录，但提示还有其他设备在线 | 有感知，但本阶段复杂度更高 | |
| 其他 | 自定义会话策略 | |

**User's choice:** 允许多设备同时保持登录  
**Notes:** 本阶段只支持当前设备退出，不做其他设备会话查看、提醒或退出所有设备。
