<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UI controls → auth-session store | 用户交互可能频繁重复触发提交、打开/关闭 modal 和 logout |
| restoring overlay → map shell | overlay 若实现错误会把空地图误导成“没有数据”或直接移除主舞台 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-12 | Spoofing | topbar identity UI | mitigate | chip 只展示 `auth-session.currentUser.username`，所有身份信息来自 bootstrap/session store，不从本地表单缓存直接渲染 |
| T-23-13 | Denial of Service | auth dialog submit | mitigate | 提交中禁用重复提交；dialog 只保留最小字段集和单一 submit CTA，避免多分支并发请求 |
| T-23-14 | Information Disclosure | restoring/map shell transition | mitigate | restoring 时保留 map shell 并覆盖柔和蒙层；logout/unauthorized 后由 23-04 store 先清空旧 records，再切匿名 UI |
</threat_model>
