<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser store → server API | 前端状态不可信，401 和 session 失效必须以服务端响应为准 |
| auth-session store → map-points store | 账号切换时旧 records 可能泄露到新会话，需要显式清场 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-09 | Information Disclosure | `auth-session` → `map-points` state handoff | mitigate | `restore/login/register/logout/handleUnauthorized` 全部先调用 `resetTravelRecordsForSessionBoundary()` 再注入新快照 |
| T-23-10 | Spoofing | web API calls | mitigate | 所有 auth/records 请求固定 `credentials: 'include'`，不在 localStorage/sessionStorage 保存 token |
| T-23-11 | Repudiation | 401/session invalid UX | mitigate | unauthorized 统一进入 `handleUnauthorized()`，落 notice 并恢复匿名状态，避免 UI 假装仍然登录 |
</threat_model>
