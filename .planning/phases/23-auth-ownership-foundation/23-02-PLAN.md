<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser cookie → auth controller | 未受信任客户端提交 `sid`，服务端必须只操作当前 cookie 对应 session |
| auth controller → database | 注册/登录会把用户输入和密码哈希写入数据库 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-03 | Information Disclosure | `User.passwordHash`, session cookie | mitigate | 只存 `argon2` 哈希；`AuthUser` 响应不返回密码字段；cookie 固定 `HttpOnly + SameSite=Lax`，生产启用 `Secure` |
| T-23-04 | Repudiation | current-device logout | mitigate | `logout` 只按当前请求 cookie 删除对应 session，并在 e2e 中断言不会误删其他设备 session |
| T-23-05 | Denial of Service | `/auth/login` 与 `/auth/register` | accept | Phase 23 不新增验证码/限流系统，但保留 DTO 验证与最小字段集，避免高复杂度 scope 扩散；后续若出现滥用再独立加 rate limiting |
</threat_model>
