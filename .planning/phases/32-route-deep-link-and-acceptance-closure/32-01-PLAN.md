**Step C — 实现 router guard（GREEN 阶段）：**

在 `apps/web/src/router/index.ts` 中：
1. 在 `/timeline` 和 `/statistics` 路由定义中添加 `meta: { requiresAuth: true }`
2. 在 `export const router` 之后添加 `router.beforeEach` 守卫：

```typescript
import { useAuthSessionStore } from '../stores/auth-session'

router.beforeEach(async (to) => {
  const authSessionStore = useAuthSessionStore()

  if (authSessionStore.status === 'restoring') {
    await authSessionStore.restoreSession()
  }

  if (to.meta.requiresAuth && authSessionStore.status !== 'authenticated') {
    return { path: '/' }
  }

  return true
})
```

**关键实现细节：**
- `useAuthSessionStore()` 必须在 `beforeEach` 内部调用（每次导航获取最新 store 实例），不能放在模块顶层
- `restoreSession()` 存在去重逻辑（`hasRestoredOnce`），重复调用安全
- 只有 `to.meta.requiresAuth === true` 的路由才触发检查，`/` 和 catch-all 不受影响
- redirect 使用 `{ path: '/' }` 对象（Vue Router 导航守卫规范返回值），不是字符串

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser URL bar → Vue Router | 用户直接输入或刷新 `/timeline` `/statistics` 时，router guard 在客户端决定访问权限 |
| Router guard → auth-session store | guard 依赖 `authSessionStore.status` 作为认证真源 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Information Disclosure | `router/index.ts` beforeEach | mitigate | 未登录访问 `/timeline` `/statistics` 在路由解析阶段 redirect 到 `/`，页面组件的 anonymous shell 不再作为路由终态暴露 |
| T-32-02 | Elevation of Privilege | `router/index.ts` route meta | mitigate | `/timeline` `/statistics` 路由标记 `meta: { requiresAuth: true }`，只有 `status === 'authenticated'` 才放行；catch-all redirect 作为额外防线 |
| T-32-03 | Tampering | `auth-session.ts` restoreSession | accept | `restoreSession()` 已有去重与错误处理逻辑（`hasRestoredOnce` / try-catch），guard 仅消费其返回的 `status`；本项目无 PII 通过 URL 泄露场景 |
</threat_model>
