**Step A — 确定托管平台：**

首先检查仓库中是否已有部署相关线索：
```bash
# 检查 package.json scripts 中的 deploy/preview 命令
grep -E "deploy|preview|build" apps/web/package.json
# 检查根目录是否有 CI/CD 配置
ls .github/workflows/ .gitlab-ci.yml vercel.json netlify.toml 2>/dev/null
# 检查 apps/server 是否承载静态文件
grep -r "static\|index.html\|serveStatic\|@fastify/static" apps/server/src/
```

根据发现确定托管平台：
- 如有 `vercel.json` 或 `.github/workflows/` 中引用 Vercel → Vercel (SPA rewrite: `"rewrites": [{"source": "/((?!api/).*)", "destination": "/index.html"}]`)
- 如有 `netlify.toml` 或 Netlify 部署脚本 → Netlify (SPA rewrite: `[[redirects]] from = "/*" to = "/index.html" status = 200`)
- 如 apps/server NestJS 承载静态文件 → 需要为 server 添加 SPA fallback 中间件
- 如无任何线索 → 默认假设为静态 CDN 部署（Vite build 产出的 `dist/` 目录）
