# Phase 32 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

Phase 32 is a route-policy and acceptance-closure phase, not a redesign phase. Preserve the existing Kawaii route shells for Timeline and Statistics, and keep the authenticated topbar menu as the only primary navigation surface for these pages. Any hosting fallback file, route guard, or documentation alignment work in this phase must be behavior-only and must not create new page chrome, banners, helper notices, or alternate page layouts.

Primary visual anchors remain:

1. `AuthTopbarControl.vue` authenticated menu with the pill actions `时间轴` and `查看统计`
2. `TimelinePageView.vue` and `StatisticsPageView.vue` page headers with the shared `返回地图` escape hatch
3. Existing state cards, overview chip rows, and `StatCard` cards

| Property | Value |
|----------|-------|
| Tool | none — custom Vue 3 + Tailwind CSS v4 utilities; no `components.json` detected, and the shadcn gate is not applicable to this Vue/Vite project |
| Preset | not applicable |
| Component library | none — use existing local Vue components and route views only |
| Icon library | none — preserve the current inline decorative icons in the topbar menu |
| Font | `Nunito Variable`, `Noto Sans SC`, `PingFang SC`, `Hiragino Sans GB`, `Microsoft YaHei`, sans-serif |

Source notes: phase scope is pre-populated from `.planning/ROADMAP.md`, `.planning/REQUIREMENTS.md`, and `32-CONTEXT.md`; current design tokens come from `apps/web/src/styles/tokens.css`; current UI patterns come from `apps/web/src/components/auth/AuthTopbarControl.vue`, `apps/web/src/views/TimelinePageView.vue`, and `apps/web/src/views/StatisticsPageView.vue`; pending acceptance details come from Phase 29/30 `HUMAN-UAT.md` and `VERIFICATION.md`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, micro-spacing inside pills |
| sm | 8px | Compact vertical spacing, `space-y-2`, small control gaps |
| md | 16px | Default card gap, default control padding, standard section spacing |
| lg | 24px | Larger section padding and header breathing room where already present |
| xl | 32px | Page-shell interior spacing and major layout gutters where already present |
| 2xl | 48px | Major section breaks only when already used by the existing shell |
| 3xl | 64px | Page-level spacing only when already used by the existing shell |

Exceptions: preserve existing inherited route-page values that are already part of the shipped UI language: `min-h-11` / 44px interactive targets, 20px panel padding and grid gaps from `p-5` / `gap-5`, 24px overview-row radius, 28px state-card radius, 32px route-shell radius, and full-pill radius for badges and buttons. Phase 32 must not introduce new spacing exceptions.

## Typography

Phase 32 must preserve the existing route-page hierarchy and keep typography stable across timeline, statistics, and topbar navigation. Any text added, renamed, or restyled by Phase 32 must map to one of the four declared roles below.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 24px | 600 | 1.2 |
| Display | 40px | 600 | 1.1 |

Typography constraints: do not add new font sizes, heavier heading weights, or alternate route-specific type scales in Phase 32. Existing shipped micro-label classes such as `text-[0.68rem]`, `text-[0.72rem]`, and `text-xs` are frozen legacy implementation details outside this contract; leave them untouched only where Phase 32 does not modify the associated UI, and do not add, restyle, or reuse them for any touched text in this phase.

## Color

Phase 32 preserves the existing soft pastel Kawaii language. Route deep-link closure and acceptance evidence must not add new success colors, deployment-status highlights, or redirect-warning treatments.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAFA` + `#FFF5F5` gradient | Page background and dominant atmospheric surface |
| Secondary (30%) | `rgba(255,255,255,0.72-0.95)`, `#fdf5ff`, `#fffaf2`, `#eef8ff` | Route shells, state cards, overview rows, menu panels, and secondary surfaces |
| Accent (10%) | `#ff78ad` | The only semantic accent. Reserve for authenticated topbar presence cues, existing CTA emphasis already shipped in the route shell, and the most prominent inline icon-pill emphasis only. |
| Accent support tints | `#f48fb1`, `#ffd7ea` | Non-semantic supporting tints only. `#f48fb1` may appear only in already-shipped icon-chip or soft CTA fill treatments. `#ffd7ea` may appear only in decorative highlight washes or low-contrast gradient fades. Neither value may be used as a standalone status, CTA, or route-state signal in Phase 32. |
| Destructive | `#c86464` | Destructive actions only; no destructive action is introduced in Phase 32 |

Accent reserved for: authenticated topbar presence cues, existing `立即登录`/auth-adjacent accents outside this phase's redirect end state, inline icon pills, and decorative warm gradients already present in the UI. Only `#ff78ad` counts as the semantic accent token in this contract. Do not use `#ff78ad`, `#f48fb1`, or `#ffd7ea` to signal deep-link success, redirect completion, deploy acceptance, or documentation closure.

## Interaction Contract

Phase 32 introduces one material behavior change to the accepted UI contract: unauthenticated direct-open or refresh on `/timeline` and `/statistics` must fail closed to `/`, not settle on the route-local anonymous cards.

| State / Interaction | Contract |
|---------------------|----------|
| Authenticated topbar navigation | Keep the username menu as the source of truth for page entry. The actions remain ordered as `时间轴` first and `查看统计` second, both using the existing pill/button styling. Clicking either action closes the menu before same-tab route navigation. |
| Authenticated direct-open | Opening `/timeline` or `/statistics` directly while authenticated lands on the existing dedicated route shell after app bootstrap. Shared topbar remains visible, map stage remains absent, and the route header with `返回地图` remains the visible escape hatch. |
| Refresh on route | Refreshing `/timeline` or `/statistics` must restore the same route shell instead of returning a 404. A brief normal restore/loading placeholder is acceptable only during app bootstrap. |
| Unauthenticated direct-open or refresh | The accepted end state is `/`. Route-local anonymous timeline/statistics cards are no longer valid end-state UI for Phase 32 deep-link acceptance. If session restore briefly runs first, redirect immediately and do not leave a stable anonymous route shell onscreen. |
| Timeline populated state | Preserve the current earliest-first record list, the summary row `共 N 条旅行记录`, the supporting chip `一次旅行一张卡片`, and the existing multi-visit card presentation. No filters, tabs, or inline editing affordances are added. |
| Timeline empty state | Preserve the current empty-state card and `去地图添加旅行` route back to `/`. Do not introduce a second empty-state layout for deployment acceptance. |
| Statistics populated state | Preserve the existing overview row and the three-card order: `总旅行次数`, `已去过地点数`, `已去过国家/地区数`. Do not add a separate completion chart, progress bar, or deploy-status badge. |
| Statistics error state | Preserve `统计数据加载失败` and `重新加载统计` exactly as the only route-local recovery path. Do not add toast, banner, or modal messaging for retry or deploy fallback. |
| Responsive acceptance | On desktop width and phone width, topbar menu buttons, route headers, overview chips, and CTA buttons remain readable, clickable, and unobstructed. No new sticky notice, redirect explainer, or acceptance ribbon may overlap them. |
| Route notation contract | All user-facing acceptance docs, QA steps, and phase artifacts use clean URLs `/timeline` and `/statistics` only. `#/timeline` and `#/statistics` are invalid and must not appear in accepted outputs. |

Accessibility invariants: preserve 44px minimum tap targets, button/link semantics, existing readable contrast, and stable focus behavior. Redirecting unauthenticated route entry must not strand keyboard focus inside a removed route shell.

## Copywriting Contract

Phase 32 keeps almost all page copy unchanged. The only copy-level contract change is that unauthenticated route-entry acceptance must rely on redirect to `/`, not on route-local anonymous copy for `/timeline` or `/statistics`.

| Element | Copy |
|---------|------|
| Primary CTA | No new primary CTA in this phase. Preserve topbar navigation labels `时间轴` and `查看统计`; preserve route-page secondary CTA `返回地图`. |
| Empty state heading | Timeline: `你的时间轴还是空白的`; Statistics: `还没有旅行数据` |
| Empty state body | Timeline: `先回到地图点亮一个去过的地点吧。保存后，这里会按时间顺序出现你的第一条旅行卡片。` Statistics: `先回到地图点亮一个去过的地点吧。保存后，这里会出现你的第一条统计数据。` |
| Error state | Statistics only: `统计数据加载失败` + `无法获取旅行统计，请稍后重试。` Timeline: no new route-local error copy is added in Phase 32. |
| Destructive confirmation | none — Phase 32 has no destructive actions |

Additional fixed copy:

- Keep route titles `时间轴` and `旅行统计`.
- Keep supporting labels `Personal travel history`, `Travel Statistics`, `共 N 条旅行记录`, `一次旅行一张卡片`, and `旅行统计概览`.
- Keep the statistics support sentence pattern `当前支持覆盖 {N} 个国家/地区。`
- Do not add a new redirect explainer, anonymous-route notice, or deploy-success copy to `/timeline` or `/statistics`.
