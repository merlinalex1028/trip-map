**Map initialization:**
- `const mapContainer = useTemplateRef<HTMLDivElement>('mapContainer')`
- `const { map, isReady } = useLeafletMap(mapContainer)`

**GeoJSON layers:**
- `const { addFeatures } = useGeoJsonLayers({ map, savedBoundaryIds, selectedBoundaryId, onBoundaryClick: handleBoundaryClick })`
- `handleBoundaryClick(boundaryId, latlng)`: Look up the boundaryId in displayPoints (via `findSavedPointByCityId` or direct search). If found in saved points, call `openSavedPointForPlaceOrStartDraft(...)` to reopen its popup directly (per D-12, no server resolve). Set `popupLatLng` for anchor positioning.

**Shard preloading (D-07, D-08):**
- Watch `isReady`: when map is ready, call `preloadSavedBoundaryShards()` which reads `savedBoundaryIds`, looks up each in manifest via `getGeometryManifestEntry`, collects unique `geometryDatasetVersion:assetKey` pairs, calls `loadGeometryShard` for each via `Promise.allSettled`, then calls `addFeatures('CN', ...)` and `addFeatures('OVERSEAS', ...)` for the loaded shard data.
- Also on map ready, load the full CN and OVERSEAS layer manifests via `listGeometryManifestEntriesByLayer('CN')` and `listGeometryManifestEntriesByLayer('OVERSEAS')`, then load each shard and add features. (This is the "on-demand" part — load all manifest shards. Since Phase 13 already produces small, pre-split shards, loading all is acceptable for v3.0 scope.)

**Map click handler (D-10, D-11):**
- `map.value.on('click', handleMapClick)` — extract `e.latlng.lat` and `e.latlng.lng`
- Port the `handleMapClick` logic from WorldMapStage.vue, replacing the normalized-point SVG coordinate math with direct lat/lng from Leaflet event
- `setPendingGeoHit({ lat, lng, x: 0, y: 0 })` — note: `x` and `y` (NormalizedPoint) are no longer meaningful in Leaflet mode. Set to 0 or compute from map container if needed for backwards compat. Check if any downstream code uses `pendingGeoHit.x/y` — if so, compute from `map.latLngToContainerPoint`.

- Call `resolveCanonicalPlace({ lat, lng })` and handle response exactly as WorldMapStage does (resolved → applyResolvedPlace, ambiguous → startPendingCanonicalSelection, failed → setInteractionNotice)
