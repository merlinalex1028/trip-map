## Typography

| Role | Size | Weight | Line Height | Source |
|------|------|--------|-------------|--------|
| Body | 16px | 400 | 1.5 | `src/styles/tokens.css` |
| Label | 14px | 600 | 1.4 | `src/styles/tokens.css` |
| Heading | 24px | 600 | 1.2 | `src/styles/tokens.css` |
| Display | 40px | 600 | 1.1 | size/line from `src/styles/tokens.css`; weight kept at 600 to preserve the existing two-weight system |

Typography rules:

- Keep the current two-weight system only: `400` for descriptive text, `600` for all labels, badges, titles, and CTA copy.
- `PosterTitleBlock` keeps `Display` for `h1`, but adds style through spacing, underline, ribbon, and color, not by introducing a decorative display font.
- Popup badge, marker labels, notices, and action labels all use `Label`; long descriptions, fallback explanations, and storage warnings use `Body`.
- Cute styling must come from framing and color, not from shrinking text or reducing line-height below the current readability baseline.

## State Semantic Contract

| State | Palette | Required cues | Component mapping |
|------|---------|---------------|-------------------|
| 未记录 | `#f5eee8` fill + `#d8c8bc` outline | neutral solid border, no glow, no looped animation | default button shells, inactive popup sections, unrecorded marker label chip |
| 已记录 | `#84c7d8` main + `#dff4f8` wash | cool outline, subtle stitched or dotted micro-texture, static presentation | saved boundary weak fill, saved marker shell, saved badge, reuse hint |
| 当前选中 | `#f48fb1` main + `#ffdce8` wash | stronger outline, layered halo, one sparkle accent at most, optional gentle pulse on marker only | selected boundary, selected marker, popup primary CTA, title underline, active badge |
| 低置信回退 | `#b8c6d9` main + `#eef3f8` wash | dashed border, cool notice strip, zero glow, explicit text notice | fallback notice, country/region continue CTA, unsupported-boundary prompt; no city boundary highlight on map |

State rules:

- Phase 8 semantics still apply: fallback-to-country/region never renders a city boundary, even in the new cute style.
- Every state must be readable by at least two cues: color plus one of outline style, badge wording, texture, or motion.
- Selected and saved states may never share the same halo color or border style.
- Decorative motifs must never override the state cue. If a sticker, bow, sparkle, or scalloped edge weakens recognition, remove it.

- Notices may soften visually, but copy must remain direct and diagnostic.

Surface rules:

- Project-wide visible surfaces prefer rounded corners by default: cards, popup shells, drawer shells, buttons, inputs, labels, and notices should all inherit a coherent rounded-corner language unless a sharper edge is required for explicit semantic contrast.
- Cute ornament density must decrease as task depth increases: title block > popup/view surfaces > drawer edit surface.
- One major surface may use either paper texture or glassy wash, not both at strong intensity.
- Full-surface character art, oversized stickers, and decorative overlays that cover map geography are forbidden in this phase.

## Motion And Ornament Budget

| Element | Motion | Limit |
|---------|--------|-------|
| Selected marker | gentle pulse or float | `1.6s` to `1.9s`, ease-in-out, scale delta not above `1.08` |
| Popup open/close | short rise + fade | `180ms` to `220ms`, ease-out |
| Boundary state change | soft fade only | `140ms` to `180ms`; no shimmer sweep across the whole polygon |
| Hover/focus feedback | scale or shadow bump | `140ms` to `160ms`, ease-out |
| Decorative sparkles | optional one-shot accent on open or select | never continuous across the full stage |

Motion rules:

- Continuous animation is allowed only on the currently selected or draft marker. Saved markers, saved boundaries, and page background stay static.
- Decorative pseudo-elements must use `pointer-events: none`.
- Respect `prefers-reduced-motion`: replace pulse and rise animations with a static color/outline state.
- The base map background never animates.

---

## Interaction Safety Contract

- Keep the current popup anchor order: `marker -> pending -> boundary`. Cute styling may not reorder interaction precedence.
- Focus-visible outlines stay explicit at `2px` with `3px` to `4px` offset; use the accent pink only for active focus, never for passive frames.
- Popup, drawer, notice banner, and storage warning must remain visually above decorative background grain.
- Marker labels, sparkles, ribbons, and popup arrow ornaments must not create additional clickable area beyond the control that owns them.
- Destructive actions stay visually separated from primary CTA and never occupy the first slot in the action row.
- Storage warning remains a high-contrast warning surface, not a cute card; reliability messaging beats style on error and corruption states.
