# Milestone v2.0: 城市主视角与可爱风格重构

**Status:** ✅ SHIPPED 2026-03-27
**Phases:** 7-10
**Total Plans:** 17

## Overview

`v2.0` 把产品主链路从“国家/地区级点位记录”升级为“城市优先选择 -> 真实城市边界高亮 -> 桌面 anchored popup 摘要交互 -> 统一可爱风格”。本 milestone 在不引入在线地理服务或后端的前提下，完成了城市候选确认、离线城市目录扩展、边界身份持久化、地图主舞台 popup 摘要交互，以及整套可爱风视觉语言收口，并在 2026-03-27 完成 desktop-only 范围对齐后通过里程碑审计。

## Phases

### Phase 7: 城市选择与兼容基线

**Goal**: 用户可以以城市为主要目标完成选择、消歧、回退和复用，同时 v1 旧点位继续可用。  
**Depends on**: Phase 6  
**Plans**: 5 plans

Plans:

- [x] 07-01: 建立稳定 `cityId`、候选排序与旧快照兼容基线
- [x] 07-02: 接入候选确认抽屉、回退 CTA 与统一复用流
- [x] 07-03: 扩充离线城市索引并把搜索升级为可用的离线检索
- [x] 07-04: 用 coverage-audited 的离线城市目录替换 43 城样例库
- [x] 07-05: 让 lookup、search 与候选抽屉全部消费扩展目录并补齐 UAT 级回归

**Details:** 让城市成为默认优先的记录目标，并通过稳定 `cityId`、中英文离线搜索、候选先行确认、国家/地区回退和 legacy 点位兼容，建立可持续扩展的城市选择基线。

### Phase 8: 城市边界高亮语义

**Goal**: 用户选中的城市和已保存城市记录都以真实边界一致呈现，并在状态切换中保持稳定。  
**Depends on**: Phase 7  
**Plans**: 4 plans

Plans:

- [x] 08-01: 建立离线边界数据契约与 `boundaryId` 持久化骨架
- [x] 08-02: 接入 store 派生边界状态与地图真实边界 overlay
- [x] 08-03: 补齐 reopen / switch / close / fallback / multi-area 回归
- [x] 08-04: 扩大当前主链路边界覆盖并补 unsupported 城市语义护栏

**Details:** 把“去过这座城市”的地图表达从单点 marker 升级为真实城市边界高亮，并补齐 reopen、switch、close、fallback 和 multi-area 城市的身份一致性与回归保护。

### Phase 9: Popup 主舞台交互

**Goal**: 用户可以在地图上下文中通过轻量 popup 完成高频操作，再按需进入完整详情或编辑。  
**Depends on**: Phase 8  
**Plans**: 5 plans

Plans:

- [x] 09-01: 拆分 summary/deep surface 契约并抽共享摘要卡
- [x] 09-02: 接入桌面 anchored popup 与地图内锚点定位
- [x] 09-03: 补桌面 anchored popup 的边缘避让、布局迁移与边界稳定性回归
- [x] 09-04: 修复 popup 长内容滚动可达性并保持 summary/deep 分工
- [x] 09-05: 将滚动边界收口到摘要卡中部内容区，稳定 header/footer 动作

**Details:** 让 popup 成为地图主舞台中的 summary surface，并通过共享 `PointSummaryCard`、edge-aware anchored popup、deep-drawer-only handoff 和长内容滚动修复，建立稳定的桌面交互节奏。

### Phase 10: 可爱风格与可读性收口

**Goal**: 地图、边界、marker、popup 和详情表面形成统一原创可爱风格，同时不牺牲可读性和交互可靠性。  
**Depends on**: Phase 9  
**Plans**: 3 plans

Plans:

- [x] 10-01: 同步 desktop-only 合同并建立全局可爱风 token / 壳层基线
- [x] 10-02: 收口地图舞台、边界与 marker 四态视觉和命中安全
- [x] 10-03: 统一 popup 与 deep drawer 表面语言并保留 summary / deep 契约

**Details:** 用暖粉/淡蓝 token、marker 四态语义、边界高亮层级和 popup/drawer 统一表面语言完成视觉重构，同时保留 hit area、安全动效和 reduced-motion 护栏。

## Milestone Summary

**Key Decisions:**

- 以城市作为主选择单位，并用 `cityId` 统一候选确认、搜索、复用和 legacy 兼容链路
- 以真实城市边界和 `boundaryId` 持久化语义替代单点 marker 的主表达
- 把 summary surface 固定在地图舞台内的 anchored popup，而 deep detail / edit 继续由 drawer 承接
- 在 Phase 10 正式把范围收口为 desktop-only，不再重新引入移动端 `peek` / safe-area 兼容壳层
- 用原创可爱风 token 收口地图、marker、popup 和 drawer，但保持四态辨识、命中安全和 reduced-motion 护栏

**Issues Resolved:**

- 关闭城市识别只停留在 demo 样本范围内的问题，扩大到可审计的离线城市目录
- 建立 `cityId -> boundaryId -> 边界高亮` 的稳定跨阶段身份链路
- 修复 popup 长内容裁切与整卡滚动问题，稳定 summary surface 的 header/content/footer 分区
- 完成 desktop-only 范围对齐，消除 `POP-03` 的文档与实现冲突，使 milestone audit 收敛为 `passed`

**Issues Deferred:**

- 城市图片、标签、游记、时间线等富内容能力留待下个 milestone
- 导入导出、分享、多设备同步和账号体系未纳入本 milestone
- 缩放/拖拽等更精细地图交互、路线规划和统计视图未纳入本 milestone

**Technical Debt Incurred:**

- `10-VALIDATION.md` 仍未收口为 compliant，Nyquist 视角下 Phase 10 仍记为 partial
- 主 bundle chunk 仍偏大，后续继续扩展地理数据或视觉资产前建议评估拆包
