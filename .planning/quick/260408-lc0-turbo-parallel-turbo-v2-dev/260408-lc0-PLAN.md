<task type="auto">
  <name>Task 1: 用 package-level Turbo 配置替代根脚本里的 `--parallel`</name>
  <files>package.json, turbo.json, apps/web/turbo.json</files>
  <action>
1. 保持根 `turbo.json` 的 `tasks.dev` 仍是共享默认值：`cache: false` 与 `persistent: true`。如果现有配置已经满足，就不要做无意义重排；仅在需要时补齐/规范化这一段。

2. 新增 `apps/web/turbo.json`，使用：
   ```json
   {
     "extends": ["//"],
     "tasks": {
       "dev": {
         "with": ["@trip-map/server#dev"]
       }
     }
   }
   ```
   目的：把“启动 web 时需要同时启动 server”这层关系显式建模到 Turbo v2 配置里，而不是依赖 CLI 的 `--parallel`。

3. 修改根 `package.json`：
   - 把 `dev` 从 `turbo run dev --parallel` 改为不含弃用参数的 Turbo v2 入口，优先以 `@trip-map/web#dev` 为入口任务，让 web 通过 `with` 拉起 `@trip-map/server#dev`。
   - `build`、`test`、`typecheck` 不要改动。
   - 处理 `dev:web` / `dev:server` 时保留语义清晰：
     - 如果沿用 `turbo run dev --filter=@trip-map/web` 会因 `with` 连带启动 server，则把 `dev:web` 改为只启动 web 包自身脚本的方式；
     - `dev:server` 继续只启动 server。
   - 不要引入 root task（如 `//#dev`）或把真正的启动逻辑搬回根脚本，遵循 Turborepo “package tasks first” 规则。

4. 避免把 `with` 写在根 `turbo.json` 的 `tasks.dev` 上。那会对所有包的 `dev` 生效，存在 `@trip-map/server#dev` 被重复调度的风险。
  </action>

    <automated>pnpm exec turbo run @trip-map/web#dev --dry=json</automated>
  </verify>
  <done>根脚本中不再出现 `--parallel`；`apps/web/turbo.json` 已声明 `tasks.dev.with = ["@trip-map/server#dev"]`；dry-run 输出能看见 web 入口及其伴随的 server dev 任务。</done>
</task>
