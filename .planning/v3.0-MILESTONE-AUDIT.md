### Critical Integration Issues

**INT-01: DTO type narrowing** (`apps/server/src/modules/records/dto/create-travel-record.dto.ts`)
- `placeKind` declared as `string`, contracts require `PlaceKind`
- `regionSystem` declared as `string`, contracts require `'CN' | 'OVERSEAS'`
- `adminType` declared as `string`, contracts require `ChinaAdminType | 'ADMIN1'`
- Root cause: `@IsString()` class-validator decorator requires `string` type, but contracts use literal union types
- Fix: Replace `@IsString()` with `@IsEnum()` or `@IsIn()` for narrowed types

**INT-02: Broken backfill import** (`apps/server/scripts/backfill-record-metadata.ts`)
- Imports `canonicalPlaceCatalogBase` from `canonical-place-fixtures.ts`
- Only `CANONICAL_RESOLVE_FIXTURES` is exported from that file
- Causes 7 typecheck errors and 6 E2E test failures (canonical-resolve: 3, record-metadata-backfill: 2, records-travel: 1)
- Fix: Import `CANONICAL_RESOLVE_FIXTURES` and adapt lookup logic
